Applications

Tools needed to build C code generated by Glade:

  • autoconf: Creates the configure script to support portability of the project. (RedHat i386 RPM)

  • automake: Creates standard Makefiles to help build the project. (RedHat i386 RPM)

Here is a list of related software:

  • libglade: loads the XML files and creates the user interfaces dynamically.

  • Glade--: converts Glade output to C++ for use with the Gtk-- bindings. (There is also gladecc which produces a different style of C++ output.)

  • GtkAda: Ada95 binding for GTK+, which includes support for Glade.

  • PiGTK: Pike binding for GTK+, which includes support for libglade.

  • PyGtk: python bindings for GTK which include some support for loading Glade XML files at runtime. A python source code generator is also being developed.

  • Perl Support: Perl modules to generate Perl source code and to create interfaces from the XML dynamically. (If you have problems with that URL try here.) The Perl modules are also available on CPAN.

  • PerlComposer: an IDE for perl/Gtk. It comes with GUI builder, editor and class viewer. In addition to its builtin interface builder it can import XML forms generated by Glade by using libglade.

  • SnapperPerl: an XML based application compiler and thin client. The compiler incorporates Glade XML, Perl scripts, and object extensions into a single XML file. The thin client provides a high level API and runs the application from the XML file.

  • eGlade: Eiffel code generator.

  • Common Lisp: There are 2 generators, one for the open source CMUCL and Clisp implementations, available here, and another for the non-free Allegro Common Lisp platform, available here.

  • Ruby bindings for libglade. See also the Ruby home page.

  • Anjuta: an Integrated Development Environment for GTK+ and GNOME.
